# ShopLens — Find the Slow Query Challenge

Welcome to the ShopLens Performance Challenge. This is an e-commerce analytics API built with Node.js, Express, and PostgreSQL using Prisma. Several key endpoints have significant performance issues that need to be diagnosed and resolved.

## Prerequisites
- Node.js (v18+)
- PostgreSQL (running locally or via Docker)

## Getting Started

1. **Clone the repository and install dependencies:**
   ```bash
   cd backend
   npm install
   ```

2. **Configure your environment:**
   Copy the example environment file and update it with your database connection string:
   ```bash
   cp ../.env.example ../.env
   ```

3. **Initialize the database:**
   Run the Prisma migration to create the schema:
   ```bash
   npx prisma migrate dev --name init
   ```

4. **Seed the database:**
   Since this is a performance challenge, the seed script will generate over 200,000 rows across various tables. This may take 2-5 minutes depending on your system:
   ```bash
   npx prisma db seed
   ```

5. **Start the server:**
   ```bash
   npm start
   ```

## The Challenge

The analytical API is live at `http://localhost:3001`. You must audit three specific endpoints that are measurably slow:

1. `GET /api/products?category=electronics`
2. `GET /api/orders/recent`
3. `GET /api/users/:id/activity` (Use a valid user ID from the database)

### Phase 1: Enable Query Logging
Open `backend/prisma.js` and update the Prisma client instantiation to enable query logging. This is required so you can capture the exact SQL generated by Prisma.

### Phase 2: Audit and Investigate
Use the captured SQL and the PostgreSQL `EXPLAIN ANALYZE` tool to investigate why each endpoint is slow. Document your findings in `QUERY_AUDIT.md`.

### Phase 3: Propose and Implement Fixes
For each bottleneck identified, implement a code or database-level fix (such as indexing, query refactoring, or schema changes). Verify that the fix resolves the problem by comparing the 'before' and 'after' `EXPLAIN ANALYZE` reports.
