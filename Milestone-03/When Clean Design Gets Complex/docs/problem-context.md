# System Context

This database supports a simple **e-commerce backend**.

The system must store:

- products
- categories
- suppliers
- inventory

The current schema was created quickly during early development.

As the system grew, problems appeared:

• Data duplication  
• Hard-to-maintain queries  
• Inconsistent updates  

Your task is to **redesign the schema using 3NF principles**.

However, remember something important:

Sometimes strict normalization increases **query complexity**.

Think like an engineer and evaluate the trade-offs.
